# Validasi Laporan Kerja Dari Tim

Panduan untuk Korlap memeriksa dan memvalidasi laporan kerja dari satgas/linmas.

## Apa itu Validasi Laporan?

Sebagai Korlap, Anda harus **memeriksa dan memvalidasi** semua laporan pekerjaan dari tim:

- **Memeriksa Deskripsi** — Apakah pekerjaan sesuai deskripsi
- **Memeriksa Foto** — Apakah ada bukti visual
- **Memeriksa Kualitas** — Apakah pekerjaan berkualitas baik
- **Setuju atau Tolak** — Approve laporan atau minta revisi

---

### Cara 1: Dari Menu

1. Buka tab **Menu** (☰)
2. Tap **"Validasi Laporan"** atau **"Laporan Menunggu Validasi"**

### Cara 2: Dari Dashboard

1. Buka dashboard web
2. Cari menu **"Laporan"** atau **"Validasi Aktivitas"**
3. Filter: **"Status: Menunggu Validasi"**

## Checklist Validasi

### Poin-Poin yang Harus Dicek

Saat validasi laporan, periksa:

#### 1. **Deskripsi Pekerjaan**
- [ ] Deskripsi jelas dan detail
- [ ] Deskripsi sesuai dengan tugas yang diberikan
- [ ] Metode kerja dijelaskan
- [ ] Hasil akhir dijelaskan

**Contoh Deskripsi Baik:**
```
"Membersihkan area depan gerbang utama dari sampah 
plastik dan daun kering. Menggunakan sapu halus dan 
pengki. Hasil sampah ditaruh di keranjang sampah 
yang disediakan. Area sudah bersih, rapi, dan siap 
untuk pengunjung."
```

**Contoh Deskripsi Buruk:**
```
"Bersih-bersih area gerbang"
```

#### 2. **Foto & Bukti**
- [ ] Minimal ada 1 foto (biasanya 2-3 foto)
- [ ] Foto jelas, terang, tidak blur
- [ ] Foto menunjukkan hasil kerja (sebelum & sesudah ideal)
- [ ] Foto tidak palsu/dipaksakan

**Foto Baik:**
- Menampilkan area sebelum & sesudah pekerjaan
- Cahaya cukup, objek terlihat jelas
- Context terlihat (bukan hanya detail kecil)

**Foto Buruk:**
- Blur, gelap, atau tidak relevan
- Tidak menunjukkan pekerjaan yang dikerjakan
- Hanya ambil foto sembarangan

#### 3. **Lokasi & Waktu**
- [ ] Lokasi sesuai dengan area yang ditugaskan
- [ ] Waktu masuk akal (bukan jam tengah malam jika pagi)
- [ ] Durasi masuk akal untuk pekerjaan

#### 4. **Tautan dengan Tugas**
- [ ] Laporan di-link dengan tugas yang sesuai (jika ada)
- [ ] Hasil laporan sesuai deskripsi tugas

## Aksi Validasi

### Pilihan 1: SETUJU (Approve)

Jika laporan **lengkap, jelas, dan berkualitas**:

1. Baca detail laporan
2. Lihat semua foto
3. Tap tombol **"[SETUJU]"** atau **"Approve"**
4. Sistem akan meminta konfirmasi:
   ```
   Setuju dengan laporan ini?
   [Ya, Setuju]  [Batal]
   ```
5. Status laporan akan jadi **"Disetujui"**
6. Laporan akan masuk record untuk evaluasi

### Pilihan 2: TOLAK (Reject)

Jika laporan **tidak memenuhi standar**:

1. Tap tombol **"[TOLAK]"** atau **"Reject"**
2. Sistem meminta alasan penolakan:
   ```
   Alasan penolakan? *
   [ ] Deskripsi tidak detail
   [ ] Foto tidak jelas/tidak relevan
   [ ] Tidak sesuai dengan tugas
   [ ] Kualitas pekerjaan tidak baik
   [ ] Lainnya: [Uraikan...]
   ```
3. Pilih alasan dan/atau tulis deskripsi
4. Tap **"Kirim"**
5. Pekerja akan dapat notifikasi penolakan dengan alasan

**Setelah Ditolak:**
- Pekerja harus buat **laporan baru** dengan perbaikan
- Pekerja bisa berdiskusi dengan Anda untuk klarifikasi

### Pilihan 3: MINTA REVISI

Jika laporan **hampir baik, tapi perlu perbaikan kecil**:

1. Tap tombol **"[MINTA REVISI]"** atau **"Request Changes"**
2. Tulis spesifikasi perubahan yang diperlukan:
   ```
   Catatan/Revisi Diminta:
   "Laporan sudah bagus, tapi foto 2 masih agak blur. 
   Bisa ambil ulang foto di area hasil pembersihan? 
   Terima kasih."
   ```
3. Tap **"Kirim"**
4. Pekerja akan dapat notifikasi dan diminta revisi
5. Setelah pekerja upload revisi, Anda review lagi

## Menulis Catatan & Feedback

### Saat Validasi, Anda Bisa Tulis Catatan

Di field **"Catatan Anda"**, tulis feedback untuk pekerja:

**Contoh Catatan Positif:**
```
"Laporan bagus! Pekerjaan terlihat berkualitas, area 
bersih dan rapi. Foto jelas menunjukkan hasil kerja. 
Lanjutkan seperti ini!"
```

**Contoh Catatan Konstruktif:**
```
"Pekerjaan sudah baik, tapi deskripsi bisa lebih detail. 
Sebutkan tools yang digunakan dan durasi waktu. Foto 
bagus, keep it up!"
```

**Jangan tulis:**
```
"Buruk"
"Gak bagus"
"Coba lagi"
```

### Tip Menulis Feedback

- **Spesifik** — Sebutkan apa yang baik/kurang
- **Konstruktif** — Arahkan ke perbaikan
- **Positif** — Apresiasi usaha, arahkan perbaikan
- **Action** — Tulis apa yang harus diperbaiki

## Offline Mode Validasi

### Validasi Saat Offline

Jika offline saat validasi:

1. Data laporan akan cached lokal
2. Anda bisa lihat foto & deskripsi
3. **Aksi approval/reject tidak bisa diproses offline**
4. Saat koneksi online, lakukan validasi lagi

## Troubleshooting Validasi

### "Foto Laporan Tidak Muncul"

**Penyebab:**
- Koneksi internet lemah saat loading foto
- File foto corrupt

**Solusi:**
1. Tap **"[LIHAT FOTO LEBIH BESAR]"** untuk reload
2. Tunggu beberapa detik untuk load
3. Jika masih gagal, bisa reject dan minta pekerja upload ulang

### "Sudah Setuju Laporan, Tapi Mau Batalkan"

**Solusi:**
- Biasanya tidak bisa batalkan approval di sistem
- Hubungi admin jika perlu reverse approval
- Sebisa mungkin hindari mistake dengan double-check sebelum approve

### "Banyak Laporan Pending, Bagaimana Manage?"

**Tips:**
1. **Filter Urgent** — Validasi tugas urgent dulu
2. **Batch Validasi** — Set waktu rutin untuk validasi (ex: setiap sore jam 16:00)
3. **Delegate** — Jika ada admin, bisa delegate beberapa validasi
4. **Komunikasi** — Minta pekerja fokus quality, bukan quantity

## Tips Validasi Efektif

:::tip
**1. Jangan Terburu-buru**
- Luangkan waktu untuk validasi proper
- Perhatikan detail foto & deskripsi
- Jangan approve hanya karena banyak

**2. Standar Konsisten**
- Terapkan standar yang sama untuk semua pekerja
- Jangan diskriminasi saat validasi
- Dokumentasikan alasan reject

**3. Feedback Konstruktif**
- Tulis apa yang baik & apa yang kurang
- Arahkan ke perbaikan
- Apresiasi usaha mereka

**4. Schedule Rutin**
- Validasi laporan setiap hari pada waktu tetap
- Jangan sampai numpuk berhari-hari
- Pekerja perlu tahu apakah laporan disetujui

**5. Komunikasi**
- Jika ada laporan buruk berulang dari satu orang, hubungi mereka
- Lakukan coaching/mentoring jika diperlukan
- Apresiasi improvement
:::
