# Docs screenshot & asset scripts

Helpers that produce the images used across the SEKAR user manual. All output goes
under [`../static/img/`](../static/img) and is referenced from the markdown pages.

| Script | What it does | When to run |
|--------|--------------|-------------|
| `capture-web.mjs` | Logs into the live web dashboard and screenshots each page → `static/img/web/*.png` | Whenever the web UI changes meaningfully |
| `gen-mobile-placeholders.mjs` | Renders branded placeholder PNGs for the mobile screens → `static/img/mobile/*.png` | After editing `mobile-screens.json` |
| `capture-mobile-auto.sh` | **Fully automated** — boots a headless emulator, installs the app, drives it with Maestro, writes real screenshots | One-command real captures (see setup below) |
| `maestro/capture.yaml` | Maestro flow: login as `satgas1` + navigate each screen + `takeScreenshot` | Edit to add/adjust captured screens |
| `capture-mobile.sh` | **Manual** capture from a device/emulator (guided adb screencap), overwriting placeholders | When you'd rather navigate by hand |
| `mobile-screens.json` | Shared manifest (filename, title, illustration, how-to-reach) for the two mobile scripts | Edit to add/rename a mobile screenshot |

Playwright is reused from the `fe/web` workspace (it owns the dependency + browser),
so install it once there if needed: `cd ../web && npx playwright install chromium`.

## Web screenshots (automated)

```bash
# From repo root. Defaults to staging; pass a base URL to target another env.
node fe/docs/scripts/capture-web.mjs
node fe/docs/scripts/capture-web.mjs http://localhost:3001
```

Logs in as `superadmin` (sees every page) plus a `staff_kecamatan` pass. Update the
credentials/usernames at the top of the script if your environment differs.

## Mobile screenshots (fully automated — `capture-mobile-auto.sh`)

Boots a headless Linux emulator, installs the published x86_64 APK, drives it with
Maestro (login + navigate), and writes real screenshots over the placeholders.

**One-time setup** (Linux Android SDK + Maestro; KVM required — `ls /dev/kvm`):

```bash
# Linux Android SDK with a google_apis x86_64 image + an AVD named "sekar"
export ANDROID_SDK_ROOT="$HOME/android-linux"
# (cmdline-tools → sdkmanager "platform-tools" "emulator" \
#   "platforms;android-34" "system-images;android-34;google_apis;x86_64")
avdmanager create avd -n sekar -k "system-images;android-34;google_apis;x86_64" -d pixel_5

# Maestro CLI
curl -Ls "https://get.maestro.mobile.dev" | bash   # installs to ~/.maestro/bin
```

**Run it:**

```bash
bash fe/docs/scripts/capture-mobile-auto.sh         # boots emu, installs app, captures
cd fe/docs && npm run build                         # preview
```

Login uses `edi_santoso` / `password123` (a staging field-worker account) — edit the
`inputText` line in `maestro/capture.yaml` to use a different user. The flow captures
the screens the **published** build exposes (Beranda / Tugas & Aktivitas / Lembur tabs,
Clock In, Profil, etc.); screens not in the current build keep their branded placeholder.

> Note: the published APK may lag the latest in-repo navigation. If the app's screens
> differ, adjust the selectors in `maestro/capture.yaml` (inspect with
> `adb shell uiautomator dump` + `adb pull /sdcard/window_dump.xml`).

## Mobile screenshots (manual device)

The field-worker screens can only be captured from a running app. Until then the docs
show branded placeholders generated by `gen-mobile-placeholders.mjs`. To replace them
with real captures:

```bash
# 1. Start an Android emulator (or plug in a device) with the SEKAR app installed.
adb devices            # confirm exactly one device is listed

# 2. Walk the guided capture — it tells you which screen to open, you press Enter.
bash fe/docs/scripts/capture-mobile.sh

# 3. Rebuild to preview.
cd fe/docs && npm run build && npm run serve
```

Each capture overwrites the same-named PNG (e.g. `static/img/mobile/absen-masuk.png`),
so the pages pick up the real screenshot with no markdown changes. To add a new mobile
screenshot, add an entry to `mobile-screens.json`, re-run the placeholder generator,
and reference `/img/mobile/<file>.png` from the relevant page.
