# Validating Work Reports From Team

<p align="center"><img src="/img/illustrations/illo-reports.svg" alt="Reports illustration" width="240" /></p>

Guide for Korlap to review and validate work reports from satgas/linmas.

## What is Report Validation?

As Korlap, you must **review and validate** all work reports from your team:

- **Review Description** — Is work matching description?
- **Review Photos** — Are there visual proofs?
- **Review Quality** — Is work quality good?
- **Approve or Reject** — Approve report or request revision

## Validation Checklist

### Points to Check

When validating a report, review:

#### 1. **Work Description**
- [ ] Description clear and detailed
- [ ] Description matches assigned task
- [ ] Work method explained
- [ ] Final result explained

**Good Description Example:**
```
"Cleaned the front gate area of plastic 
trash and dry leaves. Used soft broom and 
dustpan. Trash placed in provided basket. 
Area is now clean, neat, and ready for 
visitors."
```

**Poor Description Example:**
```
"Cleaned the gate area"
```

#### 2. **Photos & Evidence**
- [ ] At least 1 photo (usually 2-3 photos ideal)
- [ ] Photos clear, bright, not blurry
- [ ] Photos show work results (before & after ideal)
- [ ] Photos aren't fake/forced

**Good Photos:**
- Show area before & after work
- Good lighting, objects clearly visible
- Context visible (not just small details)

**Poor Photos:**
- Blurry, dark, or irrelevant
- Don't show work done
- Random shots

#### 3. **Location & Time**
- [ ] Location matches assigned area
- [ ] Time is reasonable (not midnight if morning shift)
- [ ] Duration reasonable for work

#### 4. **Task Connection**
- [ ] Report linked with correct task (if any)
- [ ] Report results match task description

## Validation Actions

### Option 1: APPROVE

If report is **complete, clear, and good quality**:

1. Read report details
2. View all photos
3. Tap **"[APPROVE]"** or **"Approve"** button
4. System will ask for confirmation:
   ```
   Approve this report?
   [Yes, Approve]  [Cancel]
   ```
5. Report status becomes **"Approved"**
6. Report goes into records for evaluation

### Option 2: REJECT

If report **doesn't meet standards**:

1. Tap **"[REJECT]"** or **"Reject"** button
2. System asks for rejection reason:
   ```
   Rejection reason? *
   [ ] Description not detailed
   [ ] Photo not clear/not relevant
   [ ] Doesn't match task
   [ ] Work quality poor
   [ ] Other: [Explain...]
   ```
3. Choose reason and/or write description
4. Tap **"Send"**
5. Worker gets notification with rejection reason

**After Rejection:**
- Worker must create **new report** with improvements
- Worker can discuss with you for clarification

### Option 3: REQUEST REVISION

If report is **almost good, but needs small improvements**:

1. Tap **"[REQUEST REVISION]"** or **"Request Changes"** button
2. Write specific changes needed:
   ```
   Notes/Revision Requested:
   "Report is good, but photo 2 is still a bit blurry. 
   Can you retake photo in the cleaned area? 
   Thank you."
   ```
3. Tap **"Send"**
4. Worker gets notification and is asked to revise
5. After worker uploads revision, you review again

## Writing Notes & Feedback

### Write Notes During Validation

In the **"Your Notes"** field, write feedback for worker:

**Positive Feedback Example:**
```
"Great report! Work looks quality, area 
clean and neat. Photos clearly show work 
results. Keep it up!"
```

**Constructive Feedback Example:**
```
"Work is good, but description could be 
more detailed. Mention tools used and time 
spent. Great photos, keep going!"
```

**Don't write:**
```
"Poor"
"Not good"
"Try again"
```

### Tips for Writing Feedback

- **Specific** — Mention what's good/missing
- **Constructive** — Guide toward improvement
- **Positive** — Appreciate effort, guide improvement
- **Action** — Write what needs improvement

## Troubleshooting Validation

### "Report Photos Not Showing"

**Causes:**
- Weak internet while loading photo
- Photo file corrupted

**Solutions:**
1. Tap **"[VIEW LARGER PHOTO]"** to reload
2. Wait a few seconds to load
3. If still fails, can reject and ask worker to re-upload

### "Already Approved Report, Want to Cancel"

**Solution:**
- Usually can't cancel approval in system
- Contact admin if need to reverse
- Try to avoid mistakes by double-checking before approve

### "Too Many Pending Reports, How to Manage?"

**Tips:**
1. **Filter Urgent** — Validate urgent tasks first
2. **Batch Validate** — Set regular validation time (ex: 4 PM daily)
3. **Delegate** — If there's admin, delegate some validation
4. **Communicate** — Ask workers to focus on quality, not quantity

## Tips for Effective Validation

:::tip
**1. Don't Rush**
- Take time to validate properly
- Pay attention to photo & description details
- Don't approve just because there's many

**2. Consistent Standards**
- Apply same standards to all workers
- Don't discriminate when validating
- Document rejection reasons

**3. Constructive Feedback**
- Write what's good & what needs improvement
- Guide toward better quality
- Appreciate their effort

**4. Regular Schedule**
- Validate reports daily at fixed time
- Don't let them pile up for days
- Workers need to know if approved

**5. Communication**
- If one worker has bad reports repeatedly, contact them
- Do coaching/mentoring if needed
- Appreciate improvement
:::
