## Executive Summary

**Headline reconciliation.** STATUS.md (May 9) claims **~70 % weighted**. `COMPLETION_STATUS.md` reads **~48 % (10/21)** — last edited two weeks earlier and stale. After verifying agent findings against the filesystem, the true number is closer to **~78 % weighted** (15 fully done, 4 partial, 2 not-started) because two items STATUS.md listed as "not started" actually exist on disk:

- **k6 load test (3-14)** — exists at `infra/loadtest/monitoring-500w.js` (218 lines, 500 VUs + 5 WS subscribers, p95 thresholds wired). STATUS.md is wrong.
- **Web pruning-requests admin pages (3-10 web)** — `fe/web/src/app/(dashboard)/pruning-requests/page.tsx` and `[id]/page.tsx` exist. STATUS.md says "not implemented".

Conversely STATUS.md is right about **3-13 CSV backfill** (5,008-row historical pruning import) — only index + task_delegations backfill migrations exist; no species/activity backfill script.

**Top 5 risks (read these even if you skip the rest):**

1. **CRITICAL — Test gates failing across all three workspaces.** Backend 78.86 % branches (gate 80 %); mobile 70.29 % stmts (floor 74 %); web 79.97 % stmts + 75.06 % funcs (gate 80 %). Two backend tests are red (`activities.service` KORLAP scope check throws `TypeError` instead of `ApiException`; `plant-due-date.service` due_soon boundary mis-classifies as overdue). Two mobile tests time out (`RequestDetailScreen`, `SubmitScreen`). Block PR merges until fixed.
2. **CRITICAL — `kecamatans` module has 0 % coverage.** Module added May 9 (ADR-033 follow-on) with controller + service + entity + DTO but no `*.spec.ts`. New backend module spec target is ≥85 %.
3. **HIGH — Schema/code mismatches that will bite production.** (a) `pruning_requests.submitted_by` FK is `RESTRICT` in migration but entity declares `CASCADE` — first attempted user delete will surface inconsistent behaviour depending on whether the path hits TypeORM or raw SQL. (b) `area_plants.status` entity allows 4 values (`ok|due_soon|overdue|unknown`) vs spec 3 (`ok|due|overdue`); no DB CHECK constraint enforces either, so drift can land silently.
4. **HIGH — `REDIS_URL` env mismatch.** `be/.env.example` says `redis://localhost:6379`; `infra/docker-compose.yml` binds Redis on `16379`. Fresh-clone setup fails to connect; in production the wrong port is more likely to silently fall back to no-Redis degraded mode.
5. **HIGH — User entity exposes everything-but-password.** `@Exclude` is applied to `password_hash` only. Every endpoint that returns relations (`task.assignee`, `pruning_request.reviewer`, `task_delegations.{from_user,to_user}`, `activity_tags.user`) leaks `email`, `phone`, `kecamatan_id`, internal flags to roles that should only see `full_name`. Mobile satgas sees admin emails today.
