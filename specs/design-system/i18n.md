# Internationalization (i18n)

**Status:** implemented (Jul 2026). Web + mobile are bilingual — **Indonesian (`id`, default) +
English (`en`)** — via `react-i18next` / `i18next`. This doc is the reference; the enforced rules
live in `CLAUDE.md` §Internationalization (root + `apps/web` + `apps/mobile`).

## Principles

- **Default `id`, alternate `en`.** Every user-facing string is localized; nothing is hardcoded.
- **API is English-canonical.** The backend returns `{ code, message }` with English `message`; the
  frontends localize by the stable `code` (never by parsing the message). See
  `apps/be/src/common/enums/api-error-codes.enum.ts` → `errors` namespace.
- **Terminology is standardized** via `specs/design-system/GLOSSARY.md` (one key per canonical term).

## Architecture

| Concern | Web (`apps/web`) | Mobile (`apps/mobile`) |
|---|---|---|
| Init | `src/lib/i18n/config.ts` (+ browser language detector) | `src/i18n/config.ts` |
| Resources | `src/lib/i18n/resources.ts` → `locales/{id,en}/*.json` | `src/i18n/resources.ts` → `locales/{id,en}/*.json` |
| Provider | `src/lib/i18n/provider.tsx` (`I18nProvider`, `LanguageSync`, `useLanguage`) | `App.tsx` bootstrap + `src/i18n/language.ts` |
| Error copy | `src/lib/i18n/error-messages.ts` → `errors` ns | `src/constants/errorCodes.ts` → `errors` ns |
| Switcher | Settings → `LanguageSwitcher` | Settings → segmented control |
| Storage | cookie + `localStorage`, synced to profile | `AsyncStorage` + `preferences` Redux slice, synced to profile |

**Language preference** is stored per-device (offline fallback) and synced to the user profile
(`users.preferred_language`, `'id'|'en'`, default `'id'`) so the choice follows the user across
devices. Applied on sign-in by `LanguageSync`. Dates use the active language for `toLocale*`/date-fns.

## Namespaces

Namespaced JSON, mirrored `id`/`en` and kept **key-portable across web + mobile**. Shared:
`common` (actions/entities/empty/nav/language), `auth`, `errors`, `validation`, `roles`, `status`,
`settings`, `navigation`. Domain: `tasks`, `activities`, `overtime`, `attendance`, `monitoring`,
`reports`, `plants`, `pruning`, `seeds`, `admin`, `home`, `menu`, `profile`, `notifications`,
`analytics`, `assets`, `schedules`, `onboarding`, `import`, `install-help`. Key style
`namespace:section.key`; interpolation via i18next `{{var}}`.

## Guardrail

Two layers, both enforced in CI:

1. **`npm run i18n:check`** (root → `scripts/i18n-parity-check.mjs`) fails when: a backend
   `ApiErrorCode` lacks copy in either platform's `errors.json`; `id`/`en` key sets diverge within any
   namespace; or the two platforms expose different namespace sets.
2. **ESLint `sekar-design/no-untranslated-literal`** (error, both `apps/web` + `apps/mobile`) flags
   hardcoded user-facing strings — JSX text and display attributes (`label`/`placeholder`/`title`/
   `aria-label`/`accessibilityLabel`/…). It skips identifiers, URLs, format tokens, brand names, and
   `<Trans>` fallbacks; test fixtures and the pre-provider `global-error.tsx` are allowlisted in the
   eslint configs. A genuine non-UI literal can be exempted with an inline
   `// eslint-disable-next-line sekar-design/no-untranslated-literal` (+ reason).

Run both (plus `npx tsc --noEmit`) before committing any UI change.

## Adding / changing UI copy

1. `t('<ns>:<key>')` in the component (`useTranslation`) or via `i18n.t` in non-component modules.
2. Add the key to **both** `id` and `en` of that namespace (id = Indonesian, en = English), identical
   key sets. Reuse shared namespaces where they fit.
3. New namespace → register in **both** `resources.ts` files.
4. Verify: `npm run i18n:check` + workspace `npx tsc --noEmit`.

## Route metadata (browser-tab / SEO titles)

Next.js `metadata` exports are static and can't read the client i18n runtime, so
route `layout.tsx`/`page.tsx` files use **`generateMetadata()`** with
`pageMetadata(key)` (`apps/web/src/lib/i18n/server-metadata.ts`), which resolves the
language from the **`sekar_lang` cookie** and returns localized `{ title, description }`
from `apps/web/src/lib/i18n/page-metadata.ts` (a server-only id/en dict, intentionally
outside `locales/` so it is not an i18next namespace). The root layout also localizes
`openGraph.title/description/locale` and the SSR `<html lang>`. Default is Indonesian
when the cookie is absent.
