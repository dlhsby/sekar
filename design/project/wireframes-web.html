Layout
  • Split: brand left (45%), form right (55%) — fixed at 1280; centers flexibly above 1600px max-width 1480.
  • Brand left holds version stamp for ops/support (which build is in prod).
  • Form right: same fields as mobile AS-1, but with Remember me (web only — refresh token TTL extends to 30d).
Behavior
  • Submit → POST /auth/login → store JWT in httpOnly cookie (web safer than localStorage).
  • If mustChangePassword: true → redirect to /auth/change-password (WB-2).
  • Else → redirect to /dashboard for Admin, /monitoring for Korlap.
Errors
  • Invalid email → inline below field (parity with AS-2).
  • Wrong password → toast top-right (parity with AS-3 but anchored top-right for web).
  • Blocked / non-employee email → toast with admin contact (parity with AS-4).

Gating
  • Triggered only when JWT carries mustChangePassword: true.

04

Web System Notes

Cross-frame conventions

Patterns shared across all web frames. These keep the surface predictable and let new pages slot in without redesigning chrome.

Layout grid
  • Sidebar 220px (collapsible to 64px @ <1280).
  • Topbar 52px — breadcrumb + ⌘K search + notifications.
  • Content area = remaining; no inner scroll on body, only cards/tables.
Loading & empty
  • Skeleton cards on first paint (300ms shimmer).
  • Empty state never shows zero-value KPIs — replaces grid with reason + 1–2 CTAs.
  • Errored card → black border + warn-soft fill + retry button.
Density
  • Default: comfortable (12.5px body).
  • Compact mode (12px) toggleable in header for 100+ row tables.
  • All clickable rows visibly hover-tinted (mint).
Keyboard
  • ⌘K opens global search; ↑↓ to navigate, Enter to jump.
  • j/k cycle table rows when table is focused.
  • Esc closes drawer / modal / dropdown.
Realtime contract
  • WebSocket: /ws/monitoring (positions), /ws/alerts (alert feed).
  • Polling fallback: 30s if WS unavailable.
  • "Last sync" timestamp shown on every realtime surface.
Responsive
  • Min: 1280×800. Below 1280 → sidebar collapses; KPI strip wraps to 3×2.
  • Below 1024 → split-screen monitoring stacks (map on top, table below).
  • Mobile web (<768) → kicks user to mobile app prompt page.
