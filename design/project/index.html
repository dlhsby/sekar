End-to-end design package for SEKAR — DLH Surabaya's worker tracking & task management system for the municipal parks & green spaces (Ruang Terbuka Hijau) department. This index walks the work in the order you should read it: tokens → wireframes → hi-fi. Mobile is the field surface (React Native, satgas/korlap/linmas); web is the supervisor & admin console (Next.js PWA). Hi-fi mobile + web share one tokens file (hifi-shared.css) that mirrors design-system.html and the generated tokens in fe/mobile/src/constants/generated/tokens.ts — same green primary, same status palette, same type, end-to-end.

How to read this

Top to bottom. Step 1 locks the system. Step 2 structures the screens. Step 3 applies the visuals.

Within each artifact, the same convention applies: masthead → anchor nav → sectioned screens with annotations. Mobile wireframes live in one single scrollable file (wireframes-mobile.html); mobile hi-fi appends new flows to hifi-mobile.html.

Conventions

Screen IDs follow FLOW-N (e.g. WL-3, AS-2, DASH-1, MON-2). Wireframes use lo-fi line-art primitives in the wf-styles.css token set. Hi-fi uses real components from the design system.

Indonesian copy is final-pass — Bahasa Indonesia, action-first, no jargon. English appears only in dev-facing notes and IDs.
